# 프롬프트 핵심 원칙

## 목표

> **"보고서만 보고도 보일러플레이트를 처음부터 다시 만들 수 있도록"**

2025년 최신 모범 사례 기반으로, 문서만 보고 처음부터 재현 가능한 Next.js 보일러플레이트 구축 가이드 작성

---

## 핵심 원칙

### 1. 검증 우선 (Verify First)

- 웹 검색 + Context7 MCP 공식 문서로 이중 검증
- 추측이 아닌 근거 기반 의사결정
- 모범 사례인지 반복 확인

### 2. 재현성 (Reproducibility)

- 문서만으로 동일한 결과 도출 가능해야 함
- 암묵지 최소화, 명시적 기록
- git commit 내역까지 구체적으로 서술
- 원문 복사 대신 핵심만 추출

### 3. 간결함 (Conciseness)

- 불필요한 내용 제거
- 핵심만 남기고 정리
- 비중 없이 사실 위주로
- 단, 근거는 유지 (간소화 ≠ 근거 삭제)

### 4. 출처 명확화 (Source Attribution)

- 무엇이 어디서 왔는지 명시 (예: cn()은 ShadCN에서 설치됨)
- 기술 선택의 이유 기록
- 병행 사용 시 근거 명시

### 5. 점진적 질문 (Progressive Questioning)

- AI가 먼저 질문하도록 유도
- 불완전한 답변보다 정확한 맥락 확보
- 작업 전 필요한 정보 사전 확인

### 6. 의미 단위 커밋 (Semantic Commits)

- 하나의 커밋 = 하나의 의미
- 변경 이력의 가독성 확보
- 파일을 의미적으로 분리하여 커밋

### 7. 공식 권장 스택 (Official Recommendations)

- 공식 권장 도구 우선 채택 (예: Biome → Lefthook)
- 불확실하면 결정 보류 후 확인

### 8. 문서 동기화 (Document Sync)

- 코드 변경 시 관련 문서 동시 업데이트
- 커밋 전 AGENTS.md, BOILERPLATE_REPORT.md 등 반영 확인
- 코드와 문서의 불일치 방지

## 요약

| 원칙           | 키워드                            |
| -------------- | --------------------------------- |
| 검증 우선      | 공식 문서, 이중 검증, 근거        |
| 재현성         | 문서화, 핵심 추출, 단계별         |
| 간결함         | 핵심만, 근거 유지                 |
| 출처 명확화    | 왜, 어디서, 근거                  |

| 의미 단위 커밋 | 분리, 가독성                      |
| 공식 권장 스택 | 권장 도구, 모범 사례              |
| 문서 동기화    | 코드-문서 일치, 커밋 전 확인      |

## 세부 의도

1. **재현성 확보** - 명령어, 설정 파일 내용을 복사-붙여넣기만으로 동일한 환경 구축
2. **모범 사례 검증** - 2025년 최신 공식 문서, 웹 검색을 통한 검증
3. **기술 선택의 명확한 근거** - 왜 이 기술을 선택했는지 문서화
4. **실용성** - 린트 오류 해결, hydration 이슈 방지 등 실제 문제 해결
5. **팀/커뮤니티 공유 목적** - 팀원, 오픈소스 커뮤니티에서 사용 가능
6. **코드-문서 일관성** - 변경 사항이 문서에 즉시 반영되어 불일치 방지

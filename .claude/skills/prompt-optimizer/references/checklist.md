# 프롬프트 자가 평가 체크리스트

프롬프트를 **실행하기 전에** 다음 체크리스트로 자가 평가하세요.

## 🎯 필수 5가지 체크

각 항목에 ✅ (있음) 또는 ❌ (없음)을 표시하세요.

### 1️⃣ 역할(Role)이 명확한가?

**체크:**
- [ ] 프롬프트가 "당신은 ~입니다" 또는 "~의 입장에서"로 시작하는가?
- [ ] 역할이 구체적인가? (예: "마케터" → "20-30대 여성을 타겟한 패션 브랜드 SNS 마케터")
- [ ] 역할과 요청이 관련이 있는가?

**나쁜 예:**
```
혼동된 역할: "의사와 프로그래머 입장에서 앱을 평가해줘"
(두 역할이 섞여 있음)
```

**좋은 예:**
```
명확한 역할: "당신은 UX 디자이너입니다.
사용자 경험 관점에서..."
```

---

### 2️⃣ 맥락(Context)이 충분한가?

**체크:**
- [ ] 상황/배경 정보가 있는가?
- [ ] 대상 고객/사용자가 명시되어 있는가?
- [ ] 이 작업이 왜 필요한지 설명되어 있는가?
- [ ] 제약 조건이 있다면 명시되어 있는가?

**나쁜 예:**
```
불충분한 맥락: "마케팅 아이디어를 5개 만들어줘"
(누구를 위한? 어떤 제품? 예산은?)
```

**좋은 예:**
```
충분한 맥락: "3개월 예산 1000만원으로
20-30대 직장인 여성을 위한
온라인 패션 브랜드의 SNS 마케팅 아이디어 5개를 제시해주세요"
```

---

### 3️⃣ 작업(Task)이 구체적인가?

**체크:**
- [ ] "좋은" "멋진" 같은 모호한 단어가 없는가?
- [ ] 정확히 무엇을 해달라는 건지 명확한가?
- [ ] 숫자나 구체적인 예시가 포함되어 있는가?

**나쁜 예:**
```
모호한 작업: "좋은 블로그 제목을 만들어줘"
(무엇이 "좋은"지 불명확)
```

**좋은 예:**
```
구체적인 작업: "호기심을 자극하는
프로그래밍 입문자용 블로그 제목 3개를 제안해주세요.
- 길이: 8-12자
- 포함: 숫자 또는 구체적인 이점"
```

---

### 4️⃣ 결과 형식(Format)이 명시되어 있는가?

**체크:**
- [ ] 결과를 어떤 형태로 받고 싶은지 명시했는가?
- [ ] 길이/분량 제한이 있는가?
- [ ] 원하는 구조가 명확한가?

**나쁜 예:**
```
형식 미지정: "회의 내용을 요약해줘"
(몇 줄? 불릿? 테이블?)
```

**좋은 예:**
```
명확한 형식:
"다음 형식으로 마크다운 테이블 작성:
- 열: 날짜 | 주제 | 결정사항 | 담당자
- 행: 3-5개 주요 항목만"
```

---

### 5️⃣ 금지 사항이 명시되어 있는가?

**체크:**
- [ ] 제외해야 할 내용이 있다면 명시했는가?
- [ ] 피해야 할 톤이나 스타일이 있다면 지적했는가?
- [ ] 피해야 할 단어나 표현이 있다면 명시했는가?

**나쁜 예:**
```
제약 없음: "회사 뉴스를 작성해줘"
(광고처럼 들릴 수도, 너무 딱딱할 수도...)
```

**좋은 예:**
```
명확한 제약:
"회사 뉴스를 작성해주세요.
- 하지 말 것: 과장된 표현, 마케팅 용어
- 톤: 중립적이고 사실 기반
- 길이: 200자 이내"
```

## 📊 빠른 평가

**몇 개가 체크되었나요?**

| 체크 개수 | 평가 | 조치 |
|----------|------|------|
| ✅✅✅✅✅ (5개) | 매우 좋음 | 바로 실행 |
| ✅✅✅✅ (4개) | 좋음 | 가능하면 보완 후 실행 |
| ✅✅✅ (3개) | 보통 | 부족한 부분 보완 필요 |
| ✅✅ (2개) 이하 | 부족 | 프롬프트 전면 개선 필요 |

## 🔧 추가 평가 항목 (고급)

### 길이와 복잡도
- [ ] 프롬프트가 너무 길지는 않은가? (500자 이내 권장)
- [ ] 요청 개수가 너무 많지는 않은가? (3-5개 권장)
- [ ] 한 번에 여러 작업을 요청하지는 않았는가?

**팁:** 복잡한 작업은 여러 프롬프트로 분리하세요.

### 예제의 질
- [ ] 예제(Few-Shot)를 사용하는가?
- [ ] 예제가 원하는 결과를 정확히 반영하는가?
- [ ] 예제가 다양한가? (좋은 예 + 피해야 할 예)

### 명확성 재확인
- [ ] 프롬프트를 다시 읽을 때 명확한가?
- [ ] 다른 사람이 읽어도 같은 의미로 이해할 것 같은가?
- [ ] 기술 용어의 정의가 명확한가?

## 💡 체크리스트 사용 팁

1. **프롬프트 작성 후** 이 체크리스트를 사용하세요
2. **실행 전**에 반드시 확인하세요
3. **결과가 만족스럽지 않으면** 다시 체크하세요 (보통 맥락 또는 구체성이 부족함)
4. **같은 유형의 작업**을 할 때는 **검증된 체크리스트를 저장**해두고 재사용하세요

---

## 📝 나만의 체크리스트 만들기

자주 하는 작업 유형별로 **커스텀 체크리스트**를 만드세요:

**예: 마케팅 콘텐츠 작성용**
- [ ] 타겟 고객층이 명시되어 있는가?
- [ ] 채널(SNS/블로그/이메일)이 명시되어 있는가?
- [ ] 원하는 톤/보이스가 정의되어 있는가?
- [ ] CTA(행동 유도)가 포함되는가?
- [ ] 브랜드 가이드라인이 반영되는가?

이렇게 하면 **같은 유형의 작업에서 더 빠르고 일관된 결과**를 얻을 수 있습니다.

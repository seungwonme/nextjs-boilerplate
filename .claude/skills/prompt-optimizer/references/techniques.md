# 프롬프팅 기법 총정리

"The Prompt Report"와 "Practitioner's Guide" 논문을 바탕으로 정리한 **58개 프롬프팅 기법**입니다.

## 📊 7가지 주요 범주

### 1. Zero-Shot (예제 없이 수행)
AI가 예제 없이 바로 작업을 수행합니다.

**기법들:**
- **Direct Prompting** - 직접 명령: "이것을 정리해줘"
- **Role-Based Prompting** - 역할 지정: "당신은 마케터입니다"
- **Instruction-Following** - 명확한 지시: "다음 3단계를 따르세요"

**언제 사용:** 명확한 작업, 일반적인 요청

---

### 2. Few-Shot (예제로 보여주기)
1-3개의 좋은 예제를 보여주고 같은 형식으로 수행하도록 유도합니다.

**기법들:**
- **In-Context Learning** - 문맥 속 학습: 예제와 함께 질문
- **One-Shot Learning** - 1개 예제 제시
- **Multi-Shot Learning** - 여러 예제 제시

**언제 사용:** 특정 형식, 톤, 스타일을 원할 때

**예시:**
```
좋은 예: "AI는 미래 기술입니다"
좋은 예: "클라우드는 데이터 저장 방식입니다"

같은 형식으로:
"머신러닝이란?"
```

---

### 3. Thought Generation (사고 과정 표현)
AI가 **단계적으로 생각하는 과정**을 명시하도록 합니다.

**기법들:**
- **Chain-of-Thought (CoT)** ⭐ - 단계별 추론
- **Step-by-Step Reasoning** - 단계적 추론
- **Decomposition** - 큰 문제를 작은 문제로 나누기
- **Tree-of-Thought** - 여러 사고 경로 탐색

**언제 사용:** 복잡한 문제, 수학, 논리 문제

**예시:**
```
"다음 문제를 단계별로 풀어주세요:
1. 먼저 상황을 분석하세요
2. 가능한 해결책을 나열하세요
3. 최적의 해결책을 선택하세요"
```

---

### 4. Ensembling (여러 방법으로 시도)
같은 문제를 다양한 각도에서 풀게 하고 최적의 답을 선택합니다.

**기법들:**
- **Self-Consistency** - 여러 번 시도하고 일관된 답 찾기
- **Multi-Perspective** - 여러 관점에서 답하기
- **Ensemble Voting** - 여러 답 중 가장 많은 투표를 받은 것 선택

**언제 사용:** 중요한 결정, 불확실성이 높을 때

---

### 5. Self-Criticism (스스로 평가하기)
AI가 **자신의 답을 평가**하고 개선하도록 유도합니다.

**기법들:**
- **Self-Evaluation** - 자기 평가
- **Revision** - 자체 수정
- **Verification** - 검증 과정 추가

**언제 사용:** 품질이 중요한 작업

**예시:**
```
"위 답이 맞나요? 다시 확인해보세요.
틀린 부분이 있으면 수정해주세요."
```

---

### 6. Decomposition (문제 분해)
큰 작업을 작은 부분으로 나누어 순차적으로 해결합니다.

**기법들:**
- **Task Decomposition** - 작업 분해
- **Modular Prompting** - 모듈식 프롬프팅
- **Pipeline Approach** - 파이프라인 방식

**언제 사용:** 복잡한 프로젝트, 다단계 작업

---

### 7. Specialized Techniques (특화 기법)

#### Prompt Chaining (프롬프트 연쇄)
여러 프롬프트를 연결하여 순차적으로 작업합니다.

#### Retrieval-Augmented Generation (RAG)
외부 정보를 참고하여 답변을 생성합니다.

```
"다음 문서를 바탕으로 답해주세요: [문서]"
```

#### Role-Play
특정 인물이나 직업의 관점에서 답합니다.

```
"당신은 수술을 받으려는 환자입니다.
의사에게 물어봐야 할 질문 5가지는?"
```

#### Perspective-Taking (관점 전환)
다양한 관점에서 같은 문제를 봅니다.

```
"비판가의 관점에서 이 계획의 문제점을 지적해주세요"
```

## 🎯 상황별 추천 기법

| 상황 | 추천 기법 | 이유 |
|------|---------|------|
| 처음, 간단한 요청 | Zero-Shot | 직설적이고 빠름 |
| 특정 형식 필요 | Few-Shot | 패턴 학습 효과 |
| 복잡한 문제 | Chain-of-Thought | 단계적 해결 |
| 창의력 필요 | Perspective-Taking | 다양한 관점 |
| 높은 품질 필요 | Self-Criticism | 자체 검증 |
| 매우 복잡한 작업 | Decomposition | 문제 단순화 |
| 결정이 중요 | Ensembling | 다중 검증 |

## 💡 기본 원칙

1. **명확성** - 무엇을 해달라는 건지 분명하게
2. **구체성** - 일반적인 표현 대신 구체적인 예시
3. **구조화** - 단계별로 정리된 지시
4. **형식 지정** - 원하는 결과 형태를 명시
5. **맥락 제공** - 왜 이 작업이 필요한지 설명

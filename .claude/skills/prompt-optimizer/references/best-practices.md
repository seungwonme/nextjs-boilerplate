# 상황별 베스트 프랙티스

**비개발자도 쉽게 따를 수 있는** 실무 가이드입니다.

## 📚 카테고리별 가이드

### 1. 텍스트 작성 (글쓰기)

#### 요청 패턴
```
당신은 [전문가/직업]입니다.
[상황]에서 [대상]을 위해
[작업]을 해주세요.

요구사항:
- 톤: [원하는 톤]
- 길이: [글자 수/줄]
- 포함할 것: [내용]
- 피할 것: [피할 내용]
```

#### 예시
```
당신은 기술 블로거입니다.
프로그래밍 입문자를 위해
"Python 함수 이해하기" 글을 작성해주세요.

요구사항:
- 톤: 친근하고 실용적
- 길이: 500-700자
- 포함: 실제 코드 예제 2개
- 피할 것: 고급 용어, 학술적 표현
```

#### 꿀팁
- ✅ 이전에 쓴 좋은 글을 예제로 제시하세요 (Few-Shot)
- ✅ 원하는 톤/스타일을 구체적으로 설명하세요
- ✅ 길이를 명시하세요 (너무 길면 불릿 포인트 제안)
- ❌ "멋진" "재미있는" 같은 모호한 표현 사용 금지

### 2. 데이터 분석 · 정리

#### 요청 패턴
```
당신은 [분석가/전문가]입니다.
다음 [데이터/정보]를 분석해주세요:
[데이터 삽입]

분석 관점:
- [분석 항목 1]
- [분석 항목 2]
- [분석 항목 3]

형식: [테이블/리스트/문단]
```

#### 예시
```
당신은 데이터 분석가입니다.
우리 지난달 SNS 통계를 분석해주세요:
- 인스타그램: 팔로워 5000, 참여율 2.3%
- 페이스북: 팔로워 3000, 참여율 1.8%
- 틱톡: 팔로워 2000, 참여율 8.5%

분석 관점:
- 채널별 강점과 약점
- 개선 전략
- 우선순위

형식: 마크다운 테이블
```

#### 꿀팁
- ✅ 데이터를 표 형태로 정렬하고 제시하세요
- ✅ 분석해야 할 항목을 명시하세요
- ✅ 결과 형식을 지정하세요 (테이블? 리스트?)
- ❌ 너무 많은 데이터를 한 번에 제시하지 마세요

### 3. 아이디어 · 전략

#### 요청 패턴
```
당신은 [전략가/기획자]입니다.
다음 [상황/문제]에 대해
[개수]개의 [아이디어 유형]을 제시해주세요.

상황: [상세 설명]
제약: [예산/시간/리소스 등]

각 아이디어는 다음 구성:
- 이름
- 설명
- 예상 효과
- 필요 리소스
```

#### 예시
```
당신은 마케팅 전략가입니다.
신제품 출시를 위해
5개의 마케팅 아이디어를 제시해주세요.

상황:
- 제품: 저가 온라인 패션
- 타겟: 20-30대 여성
- 예산: 500만원

제약: SNS 중심, 3개월 내 실행 가능

각 아이디어는 다음 구성:
- 이름
- 구체적인 실행 방법
- 예상 노출 및 클릭 수
- 필요 비용
```

#### 꿀팁
- ✅ 아이디어의 개수를 구체적으로 지정하세요
- ✅ 제약 조건을 명확히 하세요 (예산, 시간, 리소스)
- ✅ 각 아이디어의 구성 요소를 명시하세요
- ✅ 창의성을 원한다면 "제약 없이 다양한 관점에서"라고 명시하세요
- ❌ "좋은 아이디어"처럼 모호하게 요청하지 마세요

### 4. 문제 해결 · 트러블슈팅

#### 요청 패턴
```
당신은 [전문가]입니다.
다음 [문제]를 해결해주세요.

상황:
- [문제 설명]
- [이미 시도한 것]
- [제약 조건]

원하는 결과:
- [해결 목표]

해결책 형식:
1. [단계별 해결 방법]
2. [예상 결과]
3. [예방법]
```

#### 예시
```
당신은 팀 리더입니다.
우리 팀의 생산성 저하 문제를 해결해주세요.

상황:
- 팀 규모: 10명
- 문제: 작년 대비 생산성 20% 저하
- 이미 시도: 회의 시간 단축, 업무 분담 조정
- 제약: 예산 없음, 3주 내 개선

원하는 결과:
- 실질적인 생산성 개선
- 팀원 만족도 향상

해결책 형식:
1. 원인 분석
2. 단계별 개선 방안
3. 성과 측정 방법
```

#### 꿀팁
- ✅ 문제를 구체적으로 설명하세요 (수치 포함)
- ✅ 이미 시도한 것을 명시하세요 (중복 방지)
- ✅ 제약 조건을 명확히 하세요
- ✅ Chain-of-Thought를 요청하세요 ("단계별로 분석해주세요")
- ❌ 너무 일반적인 문제로 요청하지 마세요

### 5. 학습 · 설명

#### 요청 패턴
```
당신은 [분야] 튜터입니다.
[개념/주제]를 [대상]에게 설명해주세요.

설명 요구:
- 수준: [초보/중급/고급]
- 길이: [짧게/중간/상세]
- 포함: [예시/비유/비교 등]
- 피할 것: [전문 용어 등]
```

#### 예시
```
당신은 AI 튜터입니다.
"머신러닝"을 고등학생에게 설명해주세요.

설명 요구:
- 수준: 초보 (AI 개념 전혀 모름)
- 길이: 짧게 (3-5분 읽을 분량)
- 포함: 일상생활 예시 2개
- 피할 것: 수학 공식, 영문 약자 (한국어로 설명)
```

#### 꿀팁
- ✅ 대상의 수준을 명확히 하세요
- ✅ 일상적인 비유를 사용하라고 하세요
- ✅ 구체적인 예시를 포함하라고 명시하세요
- ✅ 길이를 제한하세요 (너무 상세하면 이해 어려움)
- ❌ 기술 용어를 설명 없이 사용하지 마세요

## 🔄 결과가 만족스럽지 않을 때

### 문제별 해결책

| 문제 | 원인 | 해결책 |
|------|------|--------|
| 너무 길음 | 분량 제한 없음 | "200자 이내로 수정해주세요" 명시 |
| 너무 기술적 | 용어 수준 맞지 않음 | "초등학생도 이해하게" 요청 |
| 방향이 다름 | 맥락 부족 | "~의 입장에서" 역할 추가 |
| 깊이 부족 | 요청이 일반적 | 구체적 예시 3개 추가 |
| 톤이 이상 | 원하는 톤 미지정 | "친근하지만 전문적으로" 명시 |
| 형식이 틀림 | 형식 미지정 | "마크다운 테이블로 정리" 명시 |

### 개선 요청 템플릿
```
이전 결과가 [문제점]입니다.

다음을 반영해서 수정해주세요:
1. [구체적인 개선 사항]
2. [추가하거나 빼야 할 내용]
3. [톤/형식 조정]

특히 [강조할 점]에 집중해주세요.
```

## ✅ 프롬프트 완성 체크리스트

프롬프트를 쓴 후 다음을 확인하세요:

- [ ] **역할**: "당신은 ~입니다" 명시?
- [ ] **맥락**: 상황, 대상, 배경 설명?
- [ ] **작업**: 무엇을 하는지 명확?
- [ ] **형식**: 결과 형태 지정?
- [ ] **제약**: 피해야 할 것 명시?
- [ ] **길이**: 분량 제한 있음?
- [ ] **예시**: 원하는 형태 예제 있음?

모두 체크되면 **좋은 프롬프트**입니다!

## 💡 마지막 팁

### 프롬프트 라이브러리 만들기
자주 사용하는 **프롬프트를 저장**해두세요:
- 블로그 글 작성용
- SNS 콘텐츠용
- 이메일 작성용
- 분석 요청용

같은 형식으로 다시 사용할 수 있어 시간을 절약할 수 있습니다.

### 점진적 개선
처음부터 완벽한 프롬프트를 만들려고 하지 마세요:
1. **첫 시도**: 기본 프롬프트 작성
2. **테스트**: 실행해보고 결과 확인
3. **개선**: 부족한 부분 수정
4. **반복**: 만족할 때까지 반복

이 과정이 프롬프트 스킬을 가장 빠르게 향상시킵니다!

# 실제 사용 패턴과 예제

다양한 상황에서 **실제로 사용할 수 있는 프롬프트 패턴**들입니다.

## 📝 기본 패턴: RCFT 공식

**R**ole (역할) + **C**ontext (맥락) + **F**ormat (형식) + **T**ask (작업)

```
당신은 [역할]입니다.
[상황/맥락]에서
[구체적인 작업]을 수행하세요.
결과는 [형식]으로 제시해주세요.
```

## 🎯 실무 분야별 패턴

### 1. 마케팅 · 콘텐츠

**SNS 캡션 작성:**
```
당신은 패션 브랜드의 SNS 마케터입니다.
20대 여성 고객층을 타겟으로
우리 제품의 SNS 인스타그램 캡션을 작성해주세요.
- 톤: 친근하고 세련된
- 길이: 150자 이내
- 포함: 해시태그 3개

제품: [제품 설명]
```

**블로그 제목:**
```
기술 블로그의 제목을 3개 제안해주세요.
- 독자: 프로그래밍 입문자
- 길이: 8-12자
- 포함: 숫자 또는 구체적인 예시
- 톤: 호기심을 자극하는

주제: Python 초보자를 위한 함수 사용법
```

### 2. 분석 · 전략

**시장 분석:**
```
당신은 시장 분석가입니다.
다음 정보를 바탕으로
우리 회사의 기회와 위협을 분석해주세요.

시장 상황: [정보]

분석 결과:
1. 상위 3개 기회
2. 상위 3개 위협
3. 각각에 대한 대응 전략

형식: 마크다운 리스트
```

### 3. 작문 · 편집

**문장 다듬기:**
```
당신은 전문 에디터입니다.
다음 문장을 더 명확하고 간결하게 수정해주세요.

원문:
[문장]

수정 방법:
- 불필요한 수식어 제거
- 능동형으로 변경
- 한 문장당 한 주제만 포함
```

**톤 조정:**
```
다음 텍스트를 [톤]으로 다시 작성해주세요.

원문:
[텍스트]

원하는 톤:
- 전문적이고 신뢰감 있는
- 기술 용어 최소화
- 10살 아이도 이해 가능한 수준
```

## 🔄 개선 패턴

**결과가 마음에 들지 않을 때:**

```
이전 결과가 [문제점]입니다.

다음을 반영해서 다시 작성해주세요:
1. [구체적인 개선 사항]
2. [톤/스타일 변경]
3. [형식 조정]

특히 [중점 사항]에 집중해주세요.
```

**예시:**
```
이전 결과가 너무 기술적이고 어렵습니다.

다음을 반영해서 다시 작성해주세요:
1. 기술 용어를 일상 언어로 변경
2. 비유나 예화를 추가
3. 3줄 이내로 요약

일반인도 이해할 수 있는 쉬운 설명에 집중해주세요.
```

## 🎨 창의성 기반 패턴

**브레인스토밍:**
```
당신은 창의성 전문가입니다.
제약 없이 다양한 아이디어를 생각해주세요.

주제: [주제]
대상: [누구를 위한 아이디어인가]

다음 형식으로 10개 이상의 아이디어를 제시해주세요:
- 아이디어명: [이름]
- 설명: [1-2문장]
- 장점: [주요 장점 1개]
```

**다양한 관점:**
```
다음 이슈를 세 가지 관점에서 분석해주세요.

이슈: [내용]

관점 1: 긍정적 관점
- 장점은?
- 기회는?

관점 2: 비판적 관점
- 문제점은?
- 위험은?

관점 3: 혁신적 관점
- 새로운 가능성은?
- 다른 해석은?
```

## ✅ Few-Shot 예제 패턴

**일관된 형식 유지:**

```
다음 형식으로 [작업]을 수행해주세요.

[좋은 예제 1]
입력: [입력 예시]
출력: [출력 예시]

[좋은 예제 2]
입력: [입력 예시]
출력: [출력 예시]

이제 다음에 대해 위 형식으로 답해주세요:
입력: [사용자 입력]
```

## 🚀 고급 패턴

### Chain-of-Thought 패턴
```
다음을 단계별로 해결해주세요:

문제: [문제]

풀이:
1단계: [첫 번째 분석]
2단계: [두 번째 분석]
3단계: [종합 분석]

최종 답: [답]
```

### 제약 조건 활용 패턴
```
다음 조건 내에서 [작업]을 해주세요:

반드시 포함할 것:
- [포함 사항 1]
- [포함 사항 2]

절대 포함하면 안 될 것:
- [제외 사항 1]
- [제외 사항 2]

결과: [요청 내용]
```

## 📋 체크리스트

각 프롬프트를 실행하기 전에:

- [ ] 역할이 명확한가?
- [ ] 맥락과 배경이 충분한가?
- [ ] 구체적인 예시가 포함되어 있는가?
- [ ] 형식이 명시되어 있는가?
- [ ] 금지 사항(~하지 말 것)이 명확한가?
- [ ] 분량 제한이 있는가?

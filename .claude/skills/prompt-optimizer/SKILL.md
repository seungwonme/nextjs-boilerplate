# 프롬프트 최적화 스킬

## 개요

좋은 프롬프트는 좋은 결과를 만듭니다. 이 스킬은 **초보자도 효과적인 프롬프트를 만들고 개선할 수 있도록** 구체적인 가이드와 실제 사례를 제공합니다.

세 가지 핵심 역할:
- 🎯 **처음부터**: 효과적인 프롬프트를 만드는 방법 배우기
- 🔍 **점검하기**: 만든 프롬프트의 문제점 찾기
- ⚡ **개선하기**: 더 좋은 결과를 얻기 위해 프롬프트 수정하기

## 빠른 시작 (5분)

**지금 바로 시작하려면:**

1. 당신이 원하는 결과를 **명확하게 한 문장으로 정의**하세요
   - 나쁜 예: "마케팅 아이디어를 생성해줘"
   - 좋은 예: "3개월 예산이 1000만원인 온라인 패션 브랜드의 SNS 마케팅 아이디어 5개를 제시해줘"

2. **역할 + 맥락 + 작업 + 결과 형식**으로 프롬프트 작성하세요
   ```
   당신은 [역할]입니다.
   [상황/맥락]에서
   다음을 [작업]하세요: [구체적인 요청]
   결과는 [원하는 형식]으로 제시해주세요.
   ```

3. **테스트**: 실제로 실행해보고 결과를 확인하세요

## 1️⃣ 프롬프트 작성하기

효과적인 프롬프트의 4가지 핵심 요소:

### 1. 역할(Role) 정하기
프롬프트를 시작할 때 **"당신은 ~입니다"** 로 시작하세요.

```
❌ 나쁜 예:
"영화 리뷰를 써줘"

✅ 좋은 예:
"당신은 영화 평론가입니다.
영화 '오펜하이머'의 전문적인 리뷰를 작성해주세요."
```

**역할의 효과:**
- AI가 해당 분야의 방식과 톤으로 응답하도록 유도
- 더 깊이 있고 전문적인 결과 생성

### 2. 맥락(Context) 제공하기
**왜 이 작업이 필요한지** 배경 정보를 제공하세요.

```
❌ 나쁜 예:
"회의 내용을 요약해줘"

✅ 좋은 예:
"우리 회사는 스타트업이고,
분기별 전략 회의 내용을 경영진과 팀장에게
3줄로 보고해야 합니다.
다음 회의 기록을 [역할에 맞게] 요약해주세요."
```

**맥락의 효과:**
- 중요도와 우선순위를 AI에게 전달
- 불필요한 정보를 줄이고 핵심만 추출

### 3. 작업(Task) 명확하게 하기
**정확히 무엇을 해달라는 건지** 구체적으로 설명하세요.

```
❌ 나쁜 예:
"좋은 블로그 제목을 만들어줘"

✅ 좋은 예:
"기술 블로그의 Python 튜토리얼 글 제목을 만들어주세요.
- 독자: 프로그래밍 입문자
- 톤: 친근하고 실용적
- 길이: 10자 이내"
```

**작업 명확성의 효과:**
- 방향 흔들림 없이 원하는 결과 생성
- 예상과 다른 결과를 피함

### 4. 결과 형식(Format) 지정하기
**결과를 어떤 형태로 받고 싶은지** 명시하세요.

```
❌ 나쁜 예:
"상품 설명을 작성해줘"

✅ 좋은 예:
"제품 설명을 다음 형식으로 작성해주세요:
- 제품명: [제품명]
- 핵심 기능 3개: [불릿 포인트]
- 고객 혜택: [1-2문장]
- CTA: [행동유도 문구]"
```

**형식 지정의 효과:**
- 바로 사용 가능한 결과물 획득
- 재작업 최소화

## 2️⃣ 프롬프트 분석하기 (평가)

당신이 만든 프롬프트가 **정말 좋은지 평가**하세요.

### 체크리스트 (5가지)

각 항목에 ✅ 또는 ❌를 표시하세요:

| 항목 | 확인 | 예시 |
|------|------|------|
| **역할이 명확한가?** | ✅/❌ | "당신은 UX 디자이너입니다" |
| **맥락이 충분한가?** | ✅/❌ | "이 프로젝트는 B2B SaaS입니다" |
| **요청이 구체적인가?** | ✅/❌ | "빨간색 버튼" vs "사용자 클릭율 30% 증가하는 CTA 버튼" |
| **결과 형식이 명시되어 있는가?** | ✅/❌ | "마크다운 테이블로 제시" |
| **모호한 단어가 없는가?** | ✅/❌ | "좋은" → "전문적이고 신뢰감 있는" |

✅가 3개 이상이면 좋은 프롬프트입니다.

## 3️⃣ 프롬프트 최적화하기

LLM의 결과가 **원하는 것과 다를 때** 개선하는 방법입니다.

### 상황별 해결책

**상황 1: 결과가 너무 일반적이고 깊이가 없음**
```
원인: 맥락과 구체성이 부족
해결책:
- 더 구체적인 예제 추가
- 제외해야 할 내용 명시
예: "광고가 아닌 교육적인 톤으로" / "~은 포함하지 마세요"
```

**상황 2: 결과가 방향을 벗어남**
```
원인: 역할과 맥락이 불명확
해결책:
- 역할을 더 명확하게 정의
- 금지 사항 추가
예: "마케터가 아닌 기술 전문가 관점으로" / "마케팅용어 사용 금지"
```

**상황 3: 결과가 너무 길거나 짧음**
```
원인: 분량 제한이 없음
해결책:
- 명확한 분량 지정
예: "3줄 이내", "500자 이내", "5개 항목"
```

**상황 4: 결과 품질이 일관되지 않음**
```
원인: 예제가 부족
해결책:
- 원하는 결과의 예제 제시 (Few-Shot)
예: "[좋은 예] ... [피해야 할 예] ..."
```

### 개선 프롬프트 템플릿

```
이전 결과가 마음에 들지 않습니다.
다음 점을 개선해주세요:
1. [문제점 설명] 예: "너무 기술적이고 어렵습니다"
2. [원하는 방향] 예: "8살 아이도 이해할 수 있게"
3. [추가 요청] 예: "비유나 예화를 포함해주세요"

위 내용을 반영해서 다시 작성해주세요.
```

## 4️⃣ 고급 기법

초보자를 벗어났다면 이 기법들을 시도해보세요.

### Chain-of-Thought (사고 과정 명시하기)
복잡한 문제 해결이 필요할 때 **AI가 단계별로 생각하도록** 유도합니다.

```
프롬프트에 추가하세요:
"다음 문제를 단계별로 풀어주세요:
1. 먼저 [상황 분석]
2. 그 다음 [해결 방안 도출]
3. 마지막으로 [최종 답]"

또는
"생각 과정을 보여주면서 답해주세요."
```

### Few-Shot Learning (예제로 보여주기)
같은 형식의 여러 예제를 제시하면 **AI가 패턴을 이해**합니다.

```
[좋은 예제 1]
질문: "AI란?"
답: "인공지능은 컴퓨터가 인간처럼 생각하는 기술입니다."

[좋은 예제 2]
질문: "머신러닝이란?"
답: "머신러닝은 AI가 데이터로부터 스스로 배우는 방식입니다."

위 형식으로 다음 질문에 답해주세요:
질문: "딥러닝이란?"
```

### Perspective-Taking (관점 전환하기)
**다른 관점에서 생각하도록** 요청하면 더 창의적인 결과를 얻습니다.

```
"대변인이 아닌 비판가의 입장에서 답해주세요"
"10년 뒤 미래에서 현재를 바라보면서 답해주세요"
"초등학생도 이해할 수 있게 설명해주세요"
```
